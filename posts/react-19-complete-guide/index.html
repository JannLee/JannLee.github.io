React 19 완전 가이드 — Actions, Compiler, use() 훅 실전 정복

2026년 4월 10일·읽기 5분
reactreact-19actionsreact-compilerhookstypescriptweb

React 19는 2024년 12월에 stable로 릴리즈된 메이저 버전으로, 개발자 경험(DX)과 성능을 동시에 혁신합니다. 이 가이드에서는 실무에 바로 적용할 수 있는 React 19의 핵심 기능을 TypeScript 코드 예제와 함께 살펴봅니다.

1. Actions — 비동기 상태 관리의 혁신

React 19 이전에는 서버 요청 처리 시 pending, error, success 상태를 모두 직접 관리해야 했습니다.

React 19의 Actions는 비동기 트랜지션을 자동으로 처리합니다. form의 action prop에 함수를 넘기면 됩니다.

Actions의 핵심 규칙:

  • async 함수를 action, formAction, startTransition에 전달
  • pending 상태는 자동 관리 (useFormStatus로 읽기)
  • error는 자동 error boundary로 전달
  • 낙관적 UI는 useOptimistic으로 구현

2. useActionState

useActionState는 Action의 결과(상태)를 추적하는 새 훅입니다. 기존 useFormState를 대체합니다.

useActionState의 반환값:

값타입설명
stateStateAction의 반환값 (이전 상태가 초기값)
formActionfunctionform의 action에 전달할 함수
isPendingbooleanAction 실행 중 여부

3. useFormStatus

useFormStatus는 부모 form의 제출 상태를 읽습니다. form 안의 어느 컴포넌트에서나 호출할 수 있어 제출 버튼 컴포넌트를 분리하기에 최적입니다.

주의: useFormStatus는 <form> 안에서만 작동합니다. form 외부 컴포넌트에서 호출하면 pending은 항상 false입니다.

4. useOptimistic

낙관적 UI(Optimistic UI)는 서버 응답을 기다리지 않고 즉시 UI를 업데이트하는 패턴입니다. React 19는 이를 useOptimistic 훅으로 1급 지원합니다.

5. use() 훅

use()는 React 19의 가장 유연한 훅입니다. Promise나 Context를 조건부로 읽을 수 있습니다.

Promise 읽기

조건부 Context 읽기

기존 useContext는 조건문 안에서 호출할 수 없었습니다. use()는 가능합니다.

6. React Compiler

React Compiler는 자동으로 컴포넌트와 훅을 메모이제이션합니다. useMemo, useCallback, React.memo를 수동으로 작성할 필요가 없어집니다.

Compiler는 "React의 규칙"을 준수하는 코드에서만 작동합니다. 규칙을 어기는 컴포넌트는 자동으로 건너뜁니다.

7. ref as prop

이제 forwardRef를 사용하지 않아도 됩니다. ref를 일반 prop처럼 전달할 수 있습니다.

클래스 컴포넌트의 ref는 이전과 동일하게 동작합니다.

8. Context as Provider

<Context.Provider> 대신 <Context>를 직접 Provider로 사용할 수 있습니다.

기존 <ThemeContext.Provider> 문법도 계속 동작하므로, 마이그레이션은 선택 사항입니다.

9. 문서 메타데이터 API

컴포넌트 안에서 <title>, <meta>, <link>를 직접 렌더링하면 React가 자동으로 <head>에 호이스팅합니다. 별도 라이브러리(react-helmet) 없이 사용 가능합니다.

우선순위 규칙: 동일한 태그가 여러 컴포넌트에 있으면 트리에서 더 깊은(자식) 컴포넌트의 값이 우선합니다.

10. 에셋 로딩 API

스타일시트, 스크립트, 폰트를 컴포넌트 수준에서 선언하면 React가 로딩을 최적화합니다.
